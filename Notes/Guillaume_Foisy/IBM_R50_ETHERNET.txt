---

# 🟥 Pourquoi AUTOEXEC.BAT est ignoré ?
Parce que FreeDOS 1.4 utilise un système de boot “multi‑profil” :

- **FDCONFIG.SYS** = maître  
- **FDAUTO.BAT** = remplace AUTOEXEC.BAT  
- **AUTOEXEC.BAT** = ignoré sauf si tu forces un SHELL /P *et* que tu désactives FDAUTO.BAT

Dans ton cas, FDCONFIG.SYS contient une ligne comme :

```
!AUTOEXEC=FDAUTO.BAT
```

ou

```
AUTOEXEC=FDAUTO.BAT
```

Donc FreeDOS exécute **FDAUTO.BAT**, pas AUTOEXEC.BAT.

---

# 🟦 ✔ Solution simple : ajouter ton réseau dans FDAUTO.BAT

Ouvre :

```
edit C:\FDAUTO.BAT
```

Et ajoute **à la fin** :

```
SET MTCP=C:\NET\MTCP
C:\NET\E1000PKT.COM 0x60 1 1000 FULL
C:\NET\MTCP\DHCP.EXE
```

Sauvegarde → reboot → **réseau OK au boot**.

---

# 🟩 ✔ Solution alternative : forcer FreeDOS à utiliser AUTOEXEC.BAT classique

Dans `C:\FDCONFIG.SYS`, trouve :

```
AUTOEXEC=FDAUTO.BAT
```

Remplace par :

```
AUTOEXEC=AUTOEXEC.BAT
```

OU supprime complètement la ligne.

Ensuite assure-toi que tu as :

```
SHELL=C:\COMMAND.COM /P /E:512
```

Et ton `C:\AUTOEXEC.BAT` sera exécuté normalement.

# 🟨 ✔ Solution “pro” : exécuter les deux (FDAUTO.BAT + AUTOEXEC.BAT)

```
CALL C:\AUTOEXEC.BAT
```

Ainsi :

- FreeDOS charge ses trucs  
- Puis il exécute ton AUTOEXEC.BAT  
- Tu gardes les deux mondes
