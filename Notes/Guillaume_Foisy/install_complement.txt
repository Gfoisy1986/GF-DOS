Sur du Pentium II / Pentium III / Pentium M, ça fonctionne très bien.

---

# 🟦 1. Peut‑on charger USBUHCI + USBDRIVE automatiquement au boot ?
**Oui.**  
Tu peux les charger dans `CONFIG.SYS` ou `AUTOEXEC.BAT` pour que ton USB soit monté automatiquement comme un disque DOS (ex: `D:`).

Mais…

### ✔ USBUHCI = driver USB bas niveau  
### ✔ USBDRIVE = driver de disque USB (FAT16/FAT32)

Ces deux-là peuvent être chargés automatiquement **si ton chipset UHCI est stable**.

---

# 🟩 2. Méthode recommandée (stable)  
Charge **USBUHCI** dans `CONFIG.SYS`  
Charge **USBDRIVE** dans `AUTOEXEC.BAT`

### CONFIG.SYS
```
DEVICE=C:\DOS\USBUHCI.SYS /v
```

### AUTOEXEC.BAT
```
LH C:\DOS\USBDRIVE.COM /v
```

### Résultat
- USBUHCI initialise le contrôleur USB  
- USBDRIVE monte automatiquement ta clé USB comme un disque DOS  
- Tu obtiens un lecteur `D:` ou `E:` dès le boot  

---

# 🟨 3. Méthode “full auto” (si ton chipset est 100% compatible)
Tu peux tout mettre dans `CONFIG.SYS` :

```
DEVICE=C:\DOS\USBUHCI.SYS /v
DEVICEHIGH=C:\DOS\USBDRIVE.SYS /v
```

⚠️ **Mais attention** :  
- Certains BIOS ne donnent pas le contrôle USB assez tôt  
- Certains chipsets VIA/SiS plantent si USBDRIVE est chargé trop tôt  
- Sur FreeDOS 1.4, USBDRIVE préfère être lancé après HIMEMX/JEMMEX

Donc je recommande la méthode mixte.

---

# 🟥 4. Ce qui doit rester manuel (si instable)
Si ton USB freeze, disparaît, ou refuse de monter :

Tu peux charger **USBDRIVE** manuellement après le boot :

```
USBDRIVE /v
```

Ou même recharger USBUHCI :

```
USBUHCI /stop
USBUHCI /start
```
